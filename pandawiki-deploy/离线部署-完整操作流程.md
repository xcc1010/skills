# PandaWiki 内网离线部署 —— 完整操作流程

> 适用场景:内网**不能上外网、不能用 gh CLI**。三个大离线包已切成 500MB 分卷放在本仓库 Release
> **`offline-bundles`**:https://github.com/xcc1010/skills/releases/tag/offline-bundles
>
> 全程四大步:①外网下载分卷 → ②合并+校验 → ③拷进内网放好目录 → ④起服务并配置。

---

## 大包清单(共 32 个分卷 / 约 15.1GB)

| 合并后文件 | 分卷范围 | 个数 | 用途 |
|---|---|---|---|
| `pandawiki-offline.tar.gz`(2.3G) | `.part-000` … `.part-004` | 5 | PandaWiki 本体 |
| `xinference-image.tar`(9.2G) | `.part-000` … `.part-018` | 19 | Embedding/Rerank 引擎镜像 |
| `xinference-cache.tar.gz`(3.8G) | `.part-000` … `.part-007` | 8 | bge-m3 + bge-reranker-v2-m3 权重 |

---

## 一、在能上 GitHub 的机器上:下载 32 个分卷

Release 页面(浏览器可逐个点下载):
**https://github.com/xcc1010/skills/releases/tag/offline-bundles**

### 用 curl 批量下(不需要 gh)

**Linux / Mac / Git-Bash:**
```bash
mkdir dl && cd dl
B=https://github.com/xcc1010/skills/releases/download/offline-bundles
for i in $(seq -w 0 4);  do curl -fL -O "$B/pandawiki-offline.tar.gz.part-0$i"; done
for i in $(seq -w 0 18); do curl -fL -O "$B/xinference-image.tar.part-0$i";     done
for i in $(seq -w 0 7);  do curl -fL -O "$B/xinference-cache.tar.gz.part-0$i";  done
ls | wc -l   # 应为 32
```

**Windows PowerShell:**
```powershell
mkdir dl; cd dl
$B="https://github.com/xcc1010/skills/releases/download/offline-bundles"
0..4  | % { $n="pandawiki-offline.tar.gz.part-{0:D3}" -f $_; curl.exe -fL -O "$B/$n" }
0..18 | % { $n="xinference-image.tar.part-{0:D3}"     -f $_; curl.exe -fL -O "$B/$n" }
0..7  | % { $n="xinference-cache.tar.gz.part-{0:D3}"  -f $_; curl.exe -fL -O "$B/$n" }
```

## 二、合并还原 3 个大包

**Linux / Mac / Git-Bash:**
```bash
cat pandawiki-offline.tar.gz.part-*  > pandawiki-offline.tar.gz
cat xinference-image.tar.part-*      > xinference-image.tar
cat xinference-cache.tar.gz.part-*   > xinference-cache.tar.gz
```
（或直接用本目录 `merge.sh`：`sh merge.sh`）

**Windows CMD**(用 CMD，不要用 PowerShell，`copy /b` 才对）:
```cmd
copy /b pandawiki-offline.tar.gz.part-* pandawiki-offline.tar.gz
copy /b xinference-image.tar.part-*     xinference-image.tar
copy /b xinference-cache.tar.gz.part-*  xinference-cache.tar.gz
```

---

## 三、校验 sha256（务必对上，错一位就是没合完整）

**Linux:** `sha256sum *.tar *.tar.gz`
**Windows:** `certutil -hashfile xinference-image.tar SHA256`

应等于:
```
pandawiki-offline.tar.gz : cb6a10562b4817c1d060884eeaf9d58f19d5c102544fac423d5358c0c229ccea
xinference-image.tar     : 536fc4a09890c5deaee9eac0f4ac9fd27988a6ec44ca1f97571cf1682f6d048a
xinference-cache.tar.gz  : 7a911c23ee10a46e19af3dfb2039496b02f68e14c5e1c8ea38c605130cf11177
```

> 对不上 → 是某个分卷没下全，重下那一卷再合。校验通过后，32 个 `.part-*` 可删。

## 四、拷进内网，放好目录

把合并好的 3 个大包 + 本仓库 `pandawiki-deploy/` 里的脚本一起拷进内网。目标结构:

```
pandawiki/
├─ pandawiki-offline.tar.gz        # PandaWiki 本体
├─ 部署总览.md / install.sh / docker-compose.yml / .env ...
└─ xinf-offline/
   ├─ xinference-image.tar          # 放这里
   ├─ xinference-cache.tar.gz       # 放这里
   ├─ deploy-offline.sh
   ├─ docker-compose.yml
   └─ README-XINF.md
```

> 脚本这些小文件在仓库 `pandawiki-deploy/` 里，git 下 zip 即可，不占大流量。

---

## 五、内网起服务

### A. 先起 Embedding + Rerank（Xinference）

PandaWiki 要用它的地址，所以先起这个。

```bash
cd xinf-offline
sh deploy-offline.sh
# 脚本依次做:docker load 镜像 → 解开权重缓存 → docker compose up
#            → launch bge-m3(embedding) + bge-reranker-v2-m3(rerank)
```

自测（能返回模型列表即 OK）:
```bash
curl http://localhost:9997/v1/models      # 应看到 bge-m3、bge-reranker-v2-m3
```

### B. 起 PandaWiki

```bash
tar xzf pandawiki-offline.tar.gz
cd pandawiki && sh install.sh      # 或按 部署总览.md 里的 docker compose up -d
```

### C. 控制台配置 3 个模型端点（PandaWiki 后台 → 模型设置）

| 用途 | 指向 | 模型名 / 备注 |
|---|---|---|
| Chat 对话 | 你内网大模型的 OpenAI 兼容地址 | 填你自己的 base_url + api_key |
| Embedding | `http://<Xinference机器IP>:9997/v1` | 模型名 `bge-m3` |
| Rerank | `http://<Xinference机器IP>:9997/v1` | 模型名 `bge-reranker-v2-m3` |

---

## ⚠️ 两个已知坑（部署时务必注意）

1. **Xinference 重启后模型不会自动加载**。宿主机/容器重启后，需要重跑
   `deploy-offline.sh` 里的 `xinference launch ...`，或做一个 systemd 开机脚本自动拉起。
2. **PandaWiki 里 embedding/rerank 不能复用 chat 端点**，必须分别指到 Xinference，
   否则检索/重排不生效。
