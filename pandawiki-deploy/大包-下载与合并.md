# 大离线包（Release 分卷）下载与合并

三个大包因 >2GB 无法直接放仓库,已切成 **500MB 分卷**放在本仓库 Release **`offline-bundles`**:
https://github.com/xcc1010/skills/releases/tag/offline-bundles

命名:`原文件名.part-000`、`part-001` …(按序拼回即为原文件)。

## 在一台能访问 GitHub 的机器上操作

**1. 下载全部分卷**(网页逐个点,或用 gh CLI 批量):
```bash
gh release download offline-bundles -R xcc1010/skills -D ./dl
cd ./dl
```

**2. 按文件合并还原**(同名分卷放同目录):
```bash
cat pandawiki-offline.tar.gz.part-*  > pandawiki-offline.tar.gz
cat xinference-image.tar.part-*      > xinference-image.tar
cat xinference-cache.tar.gz.part-*   > xinference-cache.tar.gz
```
Windows PowerShell 合并示例:
```powershell
cmd /c "copy /b xinference-image.tar.part-000+xinference-image.tar.part-001+... xinference-image.tar"
```
(或用本目录 `merge.sh`: `sh merge.sh`)

**3. 校验(可选)**,应与下面 sha256 一致:
- `pandawiki-offline.tar.gz` : `cb6a10562b4817c1d060884eeaf9d58f19d5c102544fac423d5358c0c229ccea`
- `xinference-image.tar`     : `536fc4a09890c5deaee9eac0f4ac9fd27988a6ec44ca1f97571cf1682f6d048a`
- `xinference-cache.tar.gz`  : `7a911c23ee10a46e19af3dfb2039496b02f68e14c5e1c8ea38c605130cf11177`

**4. 再按 `部署总览.md` 拷进内网部署。**

> 说明:合并出的 `xinference-image.tar` + `xinference-cache.tar.gz` 放进 `xinf-offline/` 目录,
> 与已入库的 `deploy-offline.sh`/`docker-compose.yml`/`README-XINF.md` 一起用。
