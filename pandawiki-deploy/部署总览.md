# PandaWiki 内网离线部署 · 总览

本目录含**两个离线包**，内网 x86-64 Linux + Docker(compose v2)即可部署，全程不联网。

| 包 | 文件 | 作用 |
|---|---|---|
| ① PandaWiki 本体 | `pandawiki-offline.tar.gz` (2.3G) | 知识库系统本体(12 个容器) |
| ② Embedding+Rerank | `xinf-offline/` (13G) | 向量 bge-m3 + 重排 bge-reranker-v2-m3(OpenAI 兼容,Xinference) |

> 为什么要 ②：PandaWiki 的对话/向量/重排都是**调 API**。内网只有对话 LLM、没有向量/重排模型，
> 所以用 ② 在内网提供 `/v1/embeddings` 和 `/v1/rerank`。三种模型三个端点，不能共用对话接口。

---

## 部署顺序(内网)

### 第 1 步：部署 Embedding+Rerank(先起,PandaWiki 要指向它)
```
cd xinf-offline
sudo sh deploy-offline.sh
```
- 建议主机 **≥ 8G 内存**(两个 bge 模型 CPU 加载)。
- 起来后:向量 `http://<本机IP>:9997/v1/embeddings` (bge-m3)、重排 `http://<本机IP>:9997/v1/rerank` (bge-reranker-v2-m3)。
- ⚠️ 主机/容器重启后模型不自动加载,需重跑 `deploy-offline.sh` 的两条 launch(可做 systemd)。

### 第 2 步：部署 PandaWiki
```
tar xzf pandawiki-offline.tar.gz   # 或按包内 README
cd pandawiki-offline && sudo sh deploy-offline.sh
```
- 控制台 `http://<本机IP>:2443`,admin 密码见 `pandawiki-offline/.env` 的 `ADMIN_PASSWORD`。

### 第 3 步：在 PandaWiki 控制台配三类模型
| 模型 | Base URL | 模型名 | API Key |
|---|---|---|---|
| 对话 Chat | 你们**内网 LLM 网关**(OpenAI 兼容) | 网关上的对话模型 | 内网 key |
| 向量 Embedding | `http://<第1步主机IP>:9997/v1` | `bge-m3` | 任意 |
| 重排 Rerank | `http://<第1步主机IP>:9997/v1` | `bge-reranker-v2-m3` | 任意 |

### 第 4 步：创建知识库 → 导入文档(URL / Sitemap / RSS / 离线文件)
